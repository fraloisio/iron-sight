Iron Sight — Operation Manual

A step-by-step guide to setting up, calibrating, and running the Iron Sight interactive shooting gallery installation. No prior knowledge of the project is assumed.

Requirements

  • A WiFi connection
    A smartphone hotspot works well and is recommended for first-time setup. Public or venue WiFi may have firewalls that interfere — if in doubt, use a hotspot.
  • A computer with a terminal Tested on MacBook
    On macOS open Terminal (Applications → Utilities → Terminal). On Windows use PowerShell or Windows Terminal. On Linux any terminal emulator will work.
  • Basic terminal familiarity
    You'll need to type a few commands and press Enter. Each command is shown in full — no prior experience is strictly required if you follow the steps carefully.
  • The game file
    shooting-gallery.html — open this in any modern browser (Chrome, Safari, Firefox). No internet connection needed.

Starting the device

The scope contains a Raspberry Pi computer that runs all the software. It needs to be powered on and connected to the same WiFi network as your computer before anything else can happen.

  1. Connect the scope to a power source using a micro USB cable. Use the port labelled PWR — not the USB data port next to it.

    📷
    IMAGE — Power port
    Photo showing the correct micro USB PWR port on the Pi Zero W2 inside or on the scope

  2. Press the yellow button on the external black box to power on the system.

    📷
    IMAGE — Yellow power button
    Photo of the yellow button on the external black box

  3. Check for the green LED inside the scope — this confirms the Pi has power and is booting.

    📷
    IMAGE — Green LED
    Photo showing the green activity LED visible through the scope body
    💡
    The LED will blink rapidly during boot. When it settles to a slow blink or stays on, the Pi is ready — this takes about 30 seconds.

  4. Wait for the ironside WiFi network to appear on your computer. This takes up to 30 seconds after the LED settles. The Pi broadcasts its own hotspot if no known network is found.

    📷
    SCREENSHOT — ironside network visible in WiFi list
    Screenshot of macOS WiFi menu showing "ironside" network

  5. Connect your computer to ironside. Password: ironsight. A portal page will open automatically in your browser (or navigate to http://10.42.0.1 manually).

    📷
    SCREENSHOT — Captive portal page
    Screenshot of the "Iron Sight — Add WiFi" portal page listing available networks

  6. Select a WiFi network from the portal list and enter its password. Both your computer and the scope will connect to this network.

    ⚠️
    Important — both the scope and your computer must be on the same network for the game to work. A smartphone hotspot is ideal. Public venue WiFi may have client isolation (devices can't talk to each other) — if the game doesn't connect, switch to a hotspot.
    ℹ️
    Once the scope connects to the new network, the ironside hotspot disappears. Reconnect your computer to the same network you just selected.
    📷

Connecting to the device

Once both your computer and the scope are on the same WiFi network, you can connect via SSH to run commands on the Pi directly.

Standard connection (WiFi)

  1. Open your terminal and type:

    ssh admin@scope.local

    When prompted for a password, type ironsight and press Enter. The password will not appear as you type — this is normal.

  2. You should see a prompt like admin@scope:~ $ — you are now inside the Pi. Every command you type here runs on the scope, not your computer.

Hostname
scope.local
Username
admin
Password
ironsight
SSH command
ssh admin@scope.local

Alternative: USB cable No WiFi needed

Connect the Pi to your computer with a data USB cable (use the USB port, not PWR). Wait 30 seconds, then:

ssh admin@10.12.194.1
ℹ️
Use 10.12.194.1 instead of scope.local for all URLs when connected via USB. Everything else is identical.

Opening a second terminal session

You will need two separate SSH sessions running simultaneously — one for calibration or detection, one to keep free for other commands. Open a new terminal window and SSH in again with the same credentials.

Calibration

Calibration teaches the system exactly where the projected screen is. It maps the camera's view to the game's coordinate space. You must calibrate at the installation site with the projector on and the IR bar in its final position. Recalibrate any time the projector, IR bar, or scope position changes.

⚠️
Before calibrating — make sure the IR bar is powered on and fixed in its final position. Any movement of the IR bar after calibration will require recalibration.

  1. In your SSH terminal, start the preview tool:

    python3 ~/preview.py

  2. On your computer, open a browser and navigate to:

    http://scope.local:8080

    You should see a live camera feed from the scope with a control panel on the right.

  3. Check the live feed for two bright white dots — these are the IR LEDs from the Wii sensor bar. If you see them, you're ready. If not:

    • Lower the Exposure slider to 1000–2000 µs to suppress ambient light
    • Check the Status indicator — it should say LOCKED when both dots are detected
    • Increase Gain slightly if the dots are too faint
    📷
    SCREENSHOT — Preview page showing two IR dots detected, status LOCKED
    Browser screenshot of scope.local:8080 with two green circles around the IR dots

  4. Once detection is stable, click SAVE PARAMS to lock in your exposure settings.

  5. Aim the rifle at the top-left calibration target shown on the preview (a circle with crosshair, 20% in from the corner). Hold steady, then click ⌜ TOP-LEFT in the browser.

    📷
    SCREENSHOT — Calibration targets visible on preview
    Screenshot showing the four corner target markers on the live camera feed

  6. Repeat for the remaining three corners in any order: ⌝ TOP-RIGHT, ⌞ BOTTOM-LEFT, ⌟ BOTTOM-RIGHT. Each button turns green when captured successfully.

  7. Once all four corners are captured, click SAVE CALIBRATION. You will see a confirmation message. The calibration is now saved to the Pi.

    💡
    Calibration persists on the Pi — you do not need to redo it every session, only when the physical setup changes.

  8. Stop the preview tool by pressing Ctrl + C in the SSH terminal.

Starting IR detection

The IR detection script is the core of the system. It reads the camera continuously, tracks the IR dots, and broadcasts the aim position and trigger events over the network for the game to receive.

  1. In your SSH terminal (make sure the preview tool is stopped first), run:

    python3 ~/ir_detect.py

  2. You should see output like:

    Params loaded from /home/admin/params.json Calibration loaded from /home/admin/calibration.json Starting IR tracker + WebSocket server... WebSocket server running on port 8765 Camera started

  3. Point the rifle at the IR bar. You should see lines like:

    cam:(160,120)/320x240 aim:(0.512,0.489)

    The aim values are normalised coordinates (0 to 1) representing where on screen the rifle is pointing.

ℹ️
Leave this terminal running. The WebSocket server on port 8765 must stay active for the game to receive position data. If you need to do something else on the Pi, open a second SSH session.

Stopping detection

Press Ctrl + C in the SSH terminal. If it doesn't respond, use:

pkill -f ir_detect.py

Starting the game

The game runs entirely in a browser — no installation, no server. It connects to the IR detection script over the local network automatically.

  1. Make sure ir_detect.py is running on the Pi (see previous section).

  2. On your computer, open shooting-gallery.html in a browser. Double-click the file, or drag it into a browser window.

  3. The game connects automatically. Within a few seconds the crosshair should respond to the rifle's aim. Point the rifle at the IR bar and move it — the crosshair should follow.

    💡
    If the crosshair doesn't move, check the Troubleshooting section. The most common cause is the computer and scope being on different networks.

  4. Pull the trigger to shoot. A hit is registered when the crosshair is over a target at the moment of firing.

ℹ️
Projecting the game — connect your computer to the projector via HDMI. Set the projector as the primary display or mirror it, then open shooting-gallery.html in full-screen mode (F11 on most browsers, or Cmd+Shift+F on macOS).

Troubleshooting

The ironside WiFi network never appears

The hotspot only activates if no known WiFi network is in range. If the Pi previously connected to a network that is still in range (e.g. your home WiFi), it will connect to that instead of broadcasting a hotspot.

Connect via USB cable (ssh admin@10.12.194.1) and remove the saved network:

nmcli connection show sudo nmcli connection delete "NETWORK-NAME"

Then reboot — the hotspot will appear after about 30 seconds.

Can't SSH in — "Connection refused" or "No route to host"

The most common cause is the computer and scope being on different networks. Confirm both are connected to the same WiFi. If using scope.local, try the IP address directly instead — run arp -a in your terminal to find the Pi's IP. As a last resort, connect via USB.

No IR dots visible in the preview — status shows "no IR bar"

Check these in order:

  • Is the IR bar plugged in and powered? It has no visible light — use a phone camera to see if it glows purple/white.
  • Is the rifle pointing at the bar? The camera must have the bar in its field of view.
  • Lower the Exposure slider — ambient light (sunlight, spotlights) can wash out the IR signal.
  • Check the Brightness stat in the panel — if it's below the Min Brightness threshold, the bar is too faint. Lower the threshold or raise Gain.
The crosshair moves in the preview but not in the game

The game connects to the WebSocket server on port 8765. Check that ir_detect.py is running (not preview.py). Open the browser console (F12) and look for WebSocket connection errors. Confirm the computer and Pi are on the same network.

The crosshair is jittery or wobbles when holding still

Open preview.py and increase the Deadzone slider — this prevents the crosshair updating for micro-movements below the threshold. A value of 0.01–0.02 is a good starting point. Also try lowering Smoothing alpha for a more damped response. Save Params when happy.

The crosshair is offset — doesn't point where the rifle is aimed

The calibration is out of date. This happens when the projector, IR bar, or scope position has changed since the last calibration. Run preview.py and redo the four-corner calibration process.

The trigger doesn't register shots

Check the physical microswitch wiring — the brown wire (COM) connects to the trigger, the green wire (NO) connects to GND. In the ir_detect.py terminal output, pulling the trigger should print Trigger fired. If it doesn't, the issue is hardware. If it does but the game doesn't respond, check the WebSocket connection.

The Pi won't shut down cleanly / is unresponsive

SSH in and run:

sudo shutdown now

Wait for the green LED to stop blinking before disconnecting power. Never pull power while the LED is active — this can corrupt the SD card.
