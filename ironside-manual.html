Iron Sight — Operation Manual

A step-by-step guide to setting up, calibrating, and running the Iron Sight interactive shooting gallery installation. No prior knowledge of the project is assumed.

How it works

Iron Sight is an interactive shooting simulation that uses infrared light tracking — the same technology found inside every TV remote control — to detect where a rifle is pointing in real time. When you aim the rifle at the projected screen and pull the trigger, the system registers a shot at the exact position you were aiming at.

There are no radio signals, no moving parts in the tracking system, and no sensors in the targets themselves. The entire tracking mechanism is self-contained inside the rifle scope.

The components

The system is made up of six physical parts that work together:

Wii Sensor Bar
A small USB-powered bar that emits two clusters of infrared light — invisible to the naked eye but visible to the camera. It is fixed near the projected screen and never moves once calibrated.
Rifle scope (the tracking unit)
A 3D-printed housing that mounts on the rifle's Picatinny rail. It contains the camera, the computer, and all the electronics. No permanent modifications to the rifle are needed.
NoIR Camera
A camera module without the infrared-blocking filter found in regular cameras. It sits at the objective end of the scope and continuously watches the screen, picking up the two glowing infrared dots from the sensor bar.
Raspberry Pi Zero W2
A small computer (roughly the size of a credit card) inside the scope that processes the camera image, calculates where the rifle is pointing, and broadcasts that information wirelessly to the game.
Trigger microswitch
A small electrical switch inside the rifle's trigger guard. When the trigger is pulled, it closes the circuit and sends a wireless shot signal to the game — registering a hit at the current aim position.
Laser pointer
An optional attachment that projects a visible dot on the screen. Useful during calibration and for helping the user understand where the invisible crosshair is pointing.

How the tracking works

The camera inside the scope looks toward the screen and sees the two infrared dots from the Wii sensor bar. By calculating the midpoint between the two dots — and how that midpoint shifts as the rifle moves — the software knows exactly which part of the screen the rifle is aimed at. A calibration step, performed once per installation, teaches the system the precise relationship between what the camera sees and where that corresponds to on the projected game screen.

All of this information travels over a local WiFi network — no internet connection required — to a browser-based game running on a nearby laptop.

💡
To check whether the Wii sensor bar is powered on, point a smartphone camera at it. The infrared LEDs will appear as bright magenta or purple dots through the phone's camera, even though they are invisible to the naked eye.

Requirements

📷
IMAGE — All components laid out on a table
Photo of rifle, scope unit, sensor bar, power bank, USB cables, laptop, and any accessories
  • A stable WiFi connection
    A smartphone hotspot works well and is the recommended option for first-time setup and testing. Public or venue WiFi can interfere with the connection for several reasons: many networks use client isolation (a security setting that prevents devices on the same network from talking to each other directly), some apply firewalls that block the type of connection the game uses (WebSocket), and others place devices in separate network zones that cannot communicate. If the game doesn't respond, switching to a smartphone hotspot almost always resolves it.
  • A computer with a terminal Tested on MacBook
    On macOS open Terminal (Applications → Utilities → Terminal). On Windows use PowerShell or Windows Terminal. On Linux any terminal emulator will work.
  • Basic terminal familiarity
    You'll need to type a few commands and press Enter. Each command is shown in full — no prior experience is strictly required if you follow the steps carefully, but some familiarity with the terminal helps.
  • The game file
    shooting-gallery.html — open this in any modern browser (Chrome, Safari, Firefox). No internet connection needed.
  • A fully charged power bank
    Used to power the rifle scope. A standard 5V USB power bank is sufficient. Check that it is fully charged before the session — the Pi draws very little power, but a flat battery mid-session will interrupt the game.
  • A USB charger or power source for the Wii sensor bar
    The sensor bar is USB-powered and draws a very small current. Any USB charger, laptop port, or power bank can power it. It must remain powered and stationary throughout the session.
  • A projector and HDMI cable
    To project the game onto the wall or screen. Connect the projector to your laptop via HDMI. Any modern projector will work.

Starting the device

The rifle scope contains a small computer that runs all the tracking software. It needs to be powered on and connected to the same WiFi network as your laptop before anything else can happen.

  1. Connect the rifle scope to the power bank using a micro USB cable. Use the port labelled PWR IN — not the USB data port next to it.

    Use the PWR port — not the USB data port next to it.

  2. Press the yellow button on the external black box to power on the system.

    Press the yellow button to power on the system.

  3. Check for a green LED inside the rifle scope — this confirms the computer has power and is booting up.

    The green LED confirms the Pi has power and is booting.
    💡
    The LED will blink rapidly during boot. When it settles to a slow blink or stays solid, the computer is ready. This takes about 30 seconds.

  4. On your laptop, open the WiFi settings and wait for a network called ironside to appear. This can take up to 30 seconds after the LED settles. If the rifle scope has been connected to a WiFi network before and that network is in range, it will connect to it automatically and the ironside hotspot will not appear — skip to the next section.

    📷

    Screenshot of macOS WiFi menu showing the "ironside" network in the list

  5. Connect your laptop to ironside. Password: ironsight. A setup page will open automatically in your browser. If it doesn't, navigate to http://10.42.0.1 manually.

    📷
    → Use image: images/wifi-b.png

  6. Select a WiFi network from the list and enter its password. This tells the rifle scope which network to join. Both the scope and your laptop must connect to that same network.

    ⚠️
    Both devices must be on the same network. A smartphone hotspot is the most reliable option. If using venue or public WiFi, check the Troubleshooting section if the game doesn't respond.
    ℹ️
    Once the rifle scope connects to the new network, the ironside hotspot disappears from your WiFi list. Reconnect your laptop to the same network you just selected.

Connecting to the device

Once both your laptop and the rifle scope are on the same WiFi network, you can open a direct connection to the computer inside the scope. This lets you run the tracking software and manage the system.

Standard connection (WiFi)

  1. Open your terminal and type the following command, then press Enter:

    ssh admin@scope.local

    When asked for a password, type ironsight and press Enter. The password will not appear on screen as you type — this is normal.

  2. You should see a prompt like admin@scope:~ $. You are now connected to the computer inside the rifle scope. Every command you type here runs on the scope, not on your laptop.

Alternative: USB cable No WiFi needed

If you have a spare USB data cable available and wireless connection is proving unreliable, connecting via USB is a robust fallback. It removes the WiFi entirely from the equation and often resolves connection issues instantly.

  1. Connect a micro USB data cable (not a charge-only cable) from your laptop to the USB port on the rifle scope — the one labelled USB, not PWR. The USB port also provides power, so do not connect the PWR cable at the same time.

    📷

  2. Wait about 30 seconds for the scope to boot and for your laptop to recognise it as a network adapter. Then connect via SSH:

    ssh admin@10.12.194.1

    Password is the same: ironsight

ℹ️
When connected via USB, use 10.12.194.1 in place of scope.local for all addresses — including the preview page (http://10.12.194.1:8080) and the game's WebSocket connection.

Opening a second terminal session

You will need two separate SSH sessions open at the same time — one running the tracking software, one free for other commands. Open a second terminal window and SSH in again with the same credentials.

Calibration

Calibration teaches the system where the projected screen is. It maps what the camera sees to the game's coordinate space, so that when the rifle points at the top-left corner of the screen, the crosshair goes to the top-left corner of the game. Calibration must be done at the installation site with the projector on and the sensor bar in its final position. Recalibrate any time the projector, sensor bar, or rifle scope position changes.

⚠️
Before calibrating — make sure the Wii sensor bar is powered on and fixed permanently in its final position. Any movement after calibration will require starting over.

  1. In your SSH terminal, start the preview and calibration tool:

    python3 ~/preview.py

  2. On your laptop, open a browser and go to:

    http://scope.local:8080

    You will see a live camera feed from inside the rifle scope, with a control panel on the right side.

  3. Look for two bright magenta or pink dots in the camera feed — these are the infrared LEDs from the sensor bar, as seen through the NoIR camera. The IR filter in the camera makes infrared light appear as a magenta or pink colour rather than white.

    If the dots are not visible, or the status panel on the right shows no IR bar, adjust the settings:

    ParameterDefaultWhat it does
    Exposure (µs) 5000 How long the camera sensor is exposed to light per frame. Lower values (try 1000–2000) darken the image and filter out ambient light, making the IR dots stand out more clearly. This is the most important setting to adjust first.
    Gain 2.0 Amplifies the sensor signal. Higher gain makes the image brighter. Increase this if the IR dots are too faint after lowering exposure. Too high causes a grainy, noisy image.
    Zoom 1.0× Digitally crops into the centre of the camera frame. 1.0 = full field of view (75°). Increase if you want to narrow the tracking area. Always recalibrate after changing zoom.
    Dilation (px) 9 Expands detected bright spots by this many pixels before analysis. Helps merge a diffuse LED glow into a single solid cluster. If both IR dots are merging into one blob, lower this value.
    Smoothing alpha 0.6 Controls how quickly the crosshair follows the rifle's movement. 1.0 = instant response but jittery. 0.1 = very smooth but slow to react. 0.5–0.7 is a good balance for most situations.
    Deadzone 0.004 The minimum movement required before the crosshair updates. Prevents the crosshair from drifting when the rifle is held steady. Increase this if the crosshair wobbles when you are not moving.
    Max dot distance (px) 192 The maximum pixel distance allowed between the two IR clusters. If two bright spots are detected but are too far apart to be the sensor bar, they are ignored. Helps reject false detections from other IR sources in the room.
    Two magenta dots detected — the status panel shows LOCKED.

  4. Once detection is stable and the status shows LOCKED, click SAVE PARAMS to store your exposure settings. They will be loaded automatically next time.

  5. Attach the laser pointer to the rifle. Aim the laser dot at the top-left calibration target visible on the preview feed — a circle with a crosshair drawn 20% in from the corner of the image. When the laser dot is on the crosshair, hold steady and click ⌜ TOP-LEFT in the browser.

    💡
    The laser pointer makes it easy to aim precisely at the corner markers. Align the laser dot with the crosshair on the preview, not with the corner of the physical screen.
    📷
    SCREENSHOT — Calibration target crosshairs visible on the preview feed
    Screenshot of the preview page with the four corner calibration crosshairs visible and one marked as captured (green)

  6. Repeat for the remaining three corners: ⌝ TOP-RIGHT, ⌞ BOTTOM-LEFT, ⌟ BOTTOM-RIGHT. Each button turns green when captured successfully.

  7. Once all four corners are captured, click SAVE CALIBRATION. You will see a confirmation message and the calibration is saved permanently to the rifle scope.

    💡
    Calibration persists across reboots — you only need to redo it when the physical setup changes.

  8. Stop the preview tool by pressing Ctrl + C in the SSH terminal.

Starting IR detection

The IR detection script is the core of the system. It continuously reads the camera, tracks the sensor bar dots, and sends aim position and trigger events over the network to the game.

  1. In your SSH terminal (make sure the preview tool is not running — press Ctrl+C if it is), start the tracker:

    python3 ~/ir_detect.py

  2. You should see confirmation messages:

    Params loaded from /home/admin/params.json Calibration loaded from /home/admin/calibration.json Starting IR tracker + WebSocket server... WebSocket server running on port 8765 Camera started

  3. Point the rifle toward the sensor bar. You should see tracking output in the terminal:

    cam:(160,120)/320x240 aim:(0.512,0.489)

    The aim values are the normalised coordinates (0 to 1) of where the rifle is currently pointing on screen.

ℹ️
Leave this terminal running. The game requires it to stay active. If you need to run other commands on the scope, open a second SSH session.

Stopping detection

Press Ctrl + C in the terminal. If it doesn't respond:

pkill -f ir_detect.py

Starting the game

The game runs entirely in a browser on your laptop — nothing to install, no internet connection needed. It connects automatically to the tracking script running on the rifle scope.

  1. Make sure ir_detect.py is running on the rifle scope (previous section).

  2. Connect your laptop to the projector via HDMI. Set the projector as your primary or mirrored display.

  3. Open shooting-gallery.html in a browser — double-click the file or drag it into an open browser window.

  4. The game connects automatically within a few seconds. The crosshair should respond immediately to the rifle's movement. Go full-screen with F11 (Windows/Linux) or Cmd+Shift+F (macOS).

    💡
    If the crosshair doesn't move, check the Troubleshooting section. The most common cause is the laptop and rifle scope being on different networks.

  5. Pull the trigger to shoot. A hit is registered when the crosshair is over a target at the moment of firing.

Troubleshooting

The ironside WiFi network never appears

The hotspot only activates when no known network is in range. If the rifle scope has previously connected to a WiFi network that is still available, it will connect to that automatically. Connect via USB cable (ssh admin@10.12.194.1) and remove the saved network:

nmcli connection show sudo nmcli connection delete "NETWORK-NAME"

Reboot the scope — the hotspot will appear after about 30 seconds.

Can't SSH in — "Connection refused" or "No route to host"

The most common cause is the laptop and scope being on different networks. Confirm both are connected to the same WiFi. Try replacing scope.local with the IP address directly — run arp -a in your terminal to find it. As a last resort, connect via USB.

No dots visible in the preview — status shows "no IR bar"
  • Is the sensor bar powered? Use a smartphone camera to check — the LEDs appear as bright magenta dots.
  • Is the rifle pointing toward the sensor bar? The camera must be able to see it.
  • Lower the Exposure slider — bright ambient light (especially sunlight or spotlights) can overwhelm the IR signal.
  • Check Brightness in the stats panel. If it is below the Min Brightness threshold, the bar is too faint — lower the threshold or raise Gain.
The crosshair moves in the preview but not in the game

The game requires ir_detect.py to be running — not preview.py. Stop the preview tool (Ctrl+C), start the tracker (python3 ~/ir_detect.py), then reload the game page. Open the browser console (F12) and check for WebSocket errors if it still doesn't work.

The crosshair wobbles or drifts when holding the rifle still

Run preview.py and increase the Deadzone slider — try 0.01 to 0.02. This prevents the crosshair from updating for movements below the threshold. Also lower Smoothing alpha for a more damped response. Click SAVE PARAMS when satisfied.

The crosshair doesn't line up with where the rifle is aimed

The calibration is out of date. This happens when the projector, sensor bar, or rifle scope has moved since the last calibration. Run preview.py and redo the four-corner calibration from scratch.

Pulling the trigger doesn't register a shot

In the ir_detect.py terminal output, pulling the trigger should print Trigger fired. If it does not, the issue is with the microswitch wiring (check the brown wire on GPIO22). If it prints but the game doesn't respond, the WebSocket connection has an issue — reload the game page.

The scope is unresponsive — SSH doesn't connect and the hotspot doesn't appear

Disconnect and reconnect the power. Wait 30–40 seconds for a full reboot. Never disconnect power while the green LED is actively blinking — this can corrupt the SD card and require a full reinstall.
